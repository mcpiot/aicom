界面概览

🖥️
整体布局

软件界面由四个主要区域组成：顶部标题栏与菜单栏、中央主显示区（含可折叠的多指令组面板）、底部控制面板，以及可选的右侧 AI 助手侧边栏。

🔴
标题栏

自定义标题栏包含红/黄/绿三个窗口控制按钮，分别对应关闭 / 最小化 / 最大化窗口。

📋
菜单栏

菜单栏从左到右依次为：MCPIOT、串口列表、串口配置、指令组发送、协议解析、TCP/UDP 转发、AI 服务配置、帮助，最右侧为主题切换与语言切换按钮。

📺
主显示区

显示串口收发的所有数据，支持时间戳、HEX 格式查看、颜色区分收发、右键上下文菜单（复制、剪切、粘贴、发送到 AI）。

⚙️
底部控制面板

包含串口参数设置、发送选项控制以及发送输入框，底部状态栏实时显示连接状态与收发字节计数。

串口连接

🔌
快速连接步骤
1
在底部控制面板的端口号下拉框中选择目标串口，或点击菜单栏「串口列表」→「刷新」更新可用端口。
2
设置波特率（默认 115200）、数据位（默认 8）、停止位（默认 1）、校验位（默认 None）。
3
点击连接按钮建立串口连接，连接成功后按钮变灰、「断开」按钮激活，状态栏显示已连接的端口名。
4
需要断开时点击断开按钮，或直接拔出设备（软件会自动检测并断开）。
⚙️
串口参数详解
参数可选值说明
波特率9600 / 19200 / 38400 / 57600 / 115200 / 230400 / 460800 / 921600 / 1M / 1.5M / 2M设备通信速率，须与目标设备一致
数据位5 / 6 / 7 / 8每帧数据的位数，通常为 8
停止位1 / 2帧结束标志位数
校验位None / Even / Odd奇偶校验方式，多数场景选 None
💡
点击菜单栏「串口配置」可在弹窗中修改波特率等参数，修改后若串口已连接，需重新连接才能生效。
📋
串口列表菜单

点击菜单栏「串口列表」展开下拉菜单，包含「刷新」按钮与当前已检测到的所有串口设备。点击端口项可直接选中该端口填入底部端口号下拉框。

数据显示

🔢
HEX 显示

勾选工具栏「HEX 显示」后，接收到的数据以十六进制格式展示，同时保留 ASCII 可读字符对照，便于分析二进制协议。

🕒
时间戳

勾选「加时间戳」后，每条接收的数据包前会附加 [HH:MM:SS.mmm] 格式的时间戳，并按超时时间自动分包显示。

💾
保存数据到文件

勾选「保存数据到文件」后，程序会弹出保存路径选择框，后续接收到的所有数据将实时追加写入该文件，取消勾选即停止保存。

📌
窗口最前

勾选「最前」可将窗口置于所有其他应用程序之上，适合同时操作设备端终端和调试工具的场景。

🖱️
右键上下文菜单

在数据显示区域右键单击，可弹出上下文菜单，支持以下操作：


操作说明
复制复制选中文本到剪贴板
剪切剪切选中文本
粘贴粘贴剪贴板内容
全选选中显示区全部内容
发送到 AI 输入框将选中文本追加到 AI 助手输入框，便于让 AI 分析串口数据
删除删除选中文本
✨
「发送到 AI 输入框」功能可将选中的协议数据直接发送给 AI 助手进行分析，是最常用的 AI 协作入口之一。
🗑️
清除与保存

工具栏「清除窗口」清空当前显示区域，不影响已保存到文件的数据。「发送文件」功能见下一节。

4

数据发送

📤
发送输入框

底部发送区域的文本框用于输入要发送的内容，点击「发送」按钮或使用快捷键发送。支持多行文本输入。

⌨️
在发送框中按 Enter 发送数据，Shift+Enter 换行输入。
🔣
HEX 发送

勾选「HEX 发送」后，输入框中的内容以十六进制字节序列解析后发送。例如输入 AA 55 01 FF 会发送 4 个字节而非 ASCII 字符串。空格和换行作为分隔符，不参与发送。

⏱️
定时发送

勾选「定时发送」并设置间隔（毫秒），发送框内容将按该间隔自动重复发送，直到取消勾选或点击清空为止。最小间隔 1ms，最大 10000ms。

↩️
加回车换行

勾选「加回车换行」后，每次发送的数据末尾自动追加 \r\n（CRLF）。许多嵌入式设备的 AT 指令需要此选项。

📁
发送文件

点击工具栏「发送文件」选择本地文件，文件内容将按二进制原样发送到串口。适用于固件升级、批量写入等场景。

⏰
时间戳与分包显示

勾选「加时间戳和分包显示」后，可设置分包超时时间（默认 20ms）。接收到数据后若在超时时间内没有新数据到达，则将已收到的数据作为一个数据包显示，并附上时间戳头。这对于分析多包协议特别有用。

多指令组发送

📝
功能介绍

多指令组发送面板（点击菜单「指令组发送」或右侧 ▶ 按钮展开）允许用户预先配置多条指令，按顺序或循环自动发送，适用于设备初始化序列、自动化测试等场景。

📊
指令表格字段
字段说明
HEX勾选后该行以 HEX 格式发送，否则以 ASCII 文本发送
HEX 字符串输入要发送的内容。双击单元格可添加注释（斜体显示），注释不会发送
点击发送点击「发送」按钮立即发送该行数据
顺序循环发送时的执行顺序编号，数字越小越先执行
延时 ms发送该条指令后等待的毫秒数，用于控制指令间隔
操作「删除」按钮移除当前行
🔁
循环发送

勾选「循环发送」后点击「发送全部」，所有行按顺序字段排列后循环执行，每行发送后等待设定的延时。点击「停止」结束循环。

📥
导入 ini

点击「导入 ini」可从标准 INI 格式文件批量导入指令配置，便于团队共享和版本管理指令集合。

↔️
拖动加宽

勾选「拖动加宽」后，可以拖动面板左边缘来调整多指令组面板的宽度，获得更大的输入区域。

➕
添加行

点击「+ 添加」在表格底部插入新的空白指令行，可无限添加。表格内容在软件重启后自动保存和恢复。

🤖
AI 助手的 set_multi_cmds 工具可由 AI 自动生成并填写整个指令序列，只需告诉 AI 你的设备协议，它会帮你完整配置所有指令和延时参数。

协议帧解析

🔬
实时帧解析器

点击菜单「协议解析」打开配置对话框。启用后，程序会实时检测接收数据中的完整协议帧，并按字段定义解析显示，无需手动截取数据。

📐
三种帧判断模式
模式适用场景必填参数
固定长度 每帧字节数固定，如 Modbus RTU 响应帧 帧头 + 帧总长度（字节数）
长度字段 帧内含长度字节，如自定义协议、CAN 帧 帧头 + 长度字节偏移 + 字段大小 + 长度含义
帧尾标识 以特定字节序列结束，如 AT 指令（\r\n） 帧头 + 帧尾（HEX）
🏷️
字段定义语法

在「字段定义」文本框中，每行定义一个字段，格式如下：

名称,偏移,长度,类型 # 示例（自定义协议）： 帧头,0,2,hex 命令字,2,1,hex 数据长度,3,1,uint8 数据内容,4,-1,hex 校验,-1,1,hex

类型说明
hex以十六进制显示原始字节，如 AA 55 01
uint8单字节无符号整数，同时显示十进制和十六进制
uint16le2 字节小端无符号整数
uint16be2 字节大端无符号整数
asciiASCII 字符串，不可打印字符显示为 .
💡
偏移或长度填 -1 具有特殊含义：偏移 -1 表示从末尾倒数，长度 -1 表示取剩余所有字节。
🤖
AI 助手的 save_frame_rule 工具可以让 AI 直接为你配置解析规则。只需告诉 AI 协议格式，它会自动填写所有参数并立即生效，无需手动配置。

TCP/UDP 转发

🌐
功能说明

点击菜单「TCP/UDP 转发」打开配置对话框。软件可同时作为 TCP 服务器、TCP 客户端或 UDP 端点，将串口数据与网络数据双向桥接转发。

🖥️
TCP 服务器模式
1
设置监听端口（默认 8888），点击「启动」。
2
外部 TCP 客户端（如手机 App、另一台 PC）连接到本机 IP + 该端口后，双方数据自动与串口互转。
3
点击「停止」关闭服务器并断开所有连接。
📡
TCP 客户端模式

填入目标服务器的 IP 地址和端口号，点击「连接」主动接入远端 TCP 服务器。连接建立后串口数据与 TCP 流双向同步，断开连接点击「断开」。

📶
UDP 转发模式

填入本地监听端口、目标 IP 和目标端口，点击「启动」。串口接收到的数据会以 UDP 包发往目标地址；收到 UDP 数据后转发给串口。适用于无连接、低延迟的数据桥接场景。

⚠️
TCP 服务器、TCP 客户端与 UDP 可同时运行，串口发出的数据会广播给所有活跃的网络端点。

AI 智能助手

🤖
打开 AI 助手

点击菜单栏最左侧「MCPIOT」按钮展开右侧 AI 助手侧边栏。再次点击可隐藏侧边栏。AI 助手支持两种服务模式，可在「AI 服务配置」中切换。

☁️
内建服务器模式

默认模式，连接 mcpiot.net 在线 AI 服务，无需任何配置，开箱即用。需要网络连接。

推荐
🏠
本地模型模式

连接本地部署的 LLM（如 Ollama、LM Studio、vLLM），在「AI 服务配置」中填入 API 地址、密钥和模型名称后生效，支持离线使用。

⚡
Agent 模式（自主操作）

在本地模式 AI 助手标题栏开启 Agent 开关后，AI 可自主调用工具完成复杂任务，无需用户手动操作软件。


工具名称功能
get_serial_log读取当前串口接收缓冲区的文本内容
get_raw_buffer读取原始字节缓冲区（HEX），适合分析二进制协议
send_to_serial向串口发送文本或 HEX 数据
open_serial_port列出可用串口或打开指定串口（含波特率等参数）
set_baud_rate动态修改已打开串口的波特率
timed_send启动或停止定时发送任务
get_multi_cmds读取多指令组面板的所有指令
set_multi_cmds向多指令组面板批量写入 AI 生成的指令序列
save_frame_rule保存并立即应用协议帧解析规则
get_frame_rule读取当前已保存的帧解析规则
calculate_checksum计算 HEX 数据的校验值（XOR / SUM8 / CRC16-Modbus 等）
start_tcp_server启动 TCP 服务器并与串口桥接
start_tcp_client以 TCP 客户端模式连接远端服务器
start_udp启动 UDP 转发
generate_docx将分析结果生成 Word 文档并保存到本地
✨
Agent 模式最多支持 8 次工具调用迭代，AI 会自主规划并逐步完成目标，例如：自动读取串口数据 → 分析协议 → 配置解析规则 → 生成分析报告。
📋
AI 服务配置
1
点击菜单「AI 服务配置」打开配置对话框。
2
选择「本地模型提供方配置」，填入 API 地址（如 http://localhost:11434/v1/chat/completions），可选填 API 密钥和模型名称。
3
点击「保存」，重新打开 AI 助手侧边栏生效。
💡
推荐模型：Qwen2.5-7B/14B（工具调用能力强）、Llama3、Gemma2。模型需支持 OpenAI 兼容 API 格式与 Function Calling。

高级功能与使用技巧

🔄
AI 协同调试工作流
1
连接串口，让设备发送几帧数据。
2
在显示区域选中接收到的数据，右键「发送到 AI 输入框」。
3
在 AI 助手输入框补充提问，如「这是一个自定义协议帧，帧头 AA 55，请分析并配置解析规则」，发送给 AI。
4
AI（Agent 模式）会自动调用 save_frame_rule 配置解析规则，之后所有接收数据均按字段解析显示。
5
如需生成调试报告，告诉 AI「生成一份 Word 文档总结调试结果」，AI 会调用 generate_docx 保存到本地。
🔢
校验码自动计算

在 AI 对话中直接描述协议并请求发送，AI 会自动调用 calculate_checksum 计算 XOR、CRC16-Modbus 等校验值，并通过 send_to_serial 直接发送完整帧。

📊
代码块快速发送

AI 回复中的代码块（如 AA 55 01 03 FF）会自动提取并填入发送框，方便一键发送。发送框会自动判断内容是否为 HEX 格式。

⚡
定时发送与 Agent

在 Agent 模式下可以告诉 AI「每隔 500ms 发送一次查询指令 01 03 00 01 00 01 HEX」，AI 会直接调用 timed_send 工具启动定时任务。

🌉
串口转网络桥接

同时开启 TCP 服务器和串口连接，可实现串口设备的网络化接入，让远程终端或移动 App 直接与串口设备通信。
